## Does K-CNAS serve my use case?

K-CNAS is designed to serve the following use cases:

### **Large Organizations**

- want to **serve many** web properties
- want to **enforce strict brand design** across their web properties
- want to **manage and empower** many teams that contribute to the web properties
- want to **reduce operational overhead** of running many web properties
- want to **reduce costs associated** with running many web properties
- want to **simplify software supply chain management** across many web properties
- want to **enforce strict security and isolation** between web properties
- want to **enforce strict compliance with regulations**
- want to **support multilingual content**

### **Small Organizations**

- want to **serve a single or few** web properties
- want to **unlock the potential of small teams**
- want to **take advantage of automated scaling**
- want to **take advantage of automated code generation**
- want to **iterate quickly** over custom business logic and frontend components without compromising security, isolation, and compliance
- want to **reduce costs associated** with running a web property
- want to **support multilingual content**

### **Commercial Entities**

- want to **serve many isolated** web properties
- want to **enforce strict security and isolation** between web properties
- want to **offer pre-built themes and structural components** to their customers
- want to **reduce time to first delivery** of new web properties
- want to **reduce operational overhead** of running many web properties
- want to **reduce costs associated** with running many web properties
- want to **enforce strict compliance with regulations**
- want to **enforce strict compliance with accessibility standards**
- want to **support multilingual content**

### **Government Agencies**

- want to **serve many web properties**
- want to **enforce strict brand design** across their web properties
- want to **manage and empower** many teams that contribute to the web properties
- want to **reduce operational overhead** of running many web properties
- want to **reduce costs associated** with running many web properties
- want to **enforce strict security and isolation** between web properties
- want to **enforce strict compliance with regulations**
- want to **enforce strict compliance with accessibility standards**
- want to **support multilingual content**
